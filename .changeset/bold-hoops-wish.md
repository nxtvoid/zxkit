Modernize the package build and authorization APIs.

- Switch the published package to ESM-only output generated by tsdown.
- Add typed navigation helpers, route metadata support, and `match: "any"` handling.
- Tighten route and permission catalog types, including wildcard permission support.
- Replace the legacy Next proxy `rules` shape with declarative `auth`, `public`, `guestOnly`, and `protected` zones.
- Fix portable client hook declaration output for consumers.
- Ensure cached authorization snapshots are never read when the user session is missing.
